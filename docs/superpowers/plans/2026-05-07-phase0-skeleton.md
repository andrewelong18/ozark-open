# Phase 0 — App Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold the Next.js + Tailwind + shadcn/ui skeleton, push it to `main`, and confirm Vercel auto-deploys successfully.

**Architecture:** `create-next-app` in the existing repo directory adds the Next.js app alongside the existing docs. A shared `<Header>` component lives in `components/header.tsx` and is imported by the root layout. Stub pages for `/dashboard` and `/bets` give Phase 1 real targets to redirect to after login.

**Tech Stack:** Next.js 14+ (App Router), TypeScript, Tailwind CSS, shadcn/ui

---

## File Map

| File | Action | Responsibility |
|---|---|---|
| `app/layout.tsx` | Replace generated | Root layout — imports Header, sets metadata |
| `app/page.tsx` | Replace generated | Landing page |

| `.env.local.example` | Create | Documents required env vars |
